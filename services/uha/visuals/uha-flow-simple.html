1 Workflow Context
Workflow
UHA | Post-Purchase Nurture → Challenge Upsell
Trigger
GHL checkout at callmedoctorp.com/ultimate-hormone-assessment-checkout captures email/first-name/payment → tag uha_purchased.
Entry
Contact has uha_purchased AND NOT uha_pitch2_sent (prevents re-entry after full sequence).
Delivery
Interact quiz embedded at callmedoctorp.com/the-ultimate-hormone-assessment-quiz (48Q ≈ 10 min) + GHL email sequence. No Kajabi course — no password setup. Quiz completion webhook fires back to GHL. Note: the Interact quiz will need to be duplicated to match the new automation flow.
Purpose
Deliver hormone results + 10-day nurture that pitches the 7-Day Challenge twice (Day 6, Day 10). Both result variants currently point to /hormone-assessment-result (placeholder — separate ED and Balanced pages pending), each with a soft Challenge CTA.
2 Key GHL Tags
  • uha_purchased — primary entry flag.
  • uha_quiz_started — on quiz page view (Interact pixel/webhook).
  • uha_quiz_done — on quiz submit; gates UHA-2 Results email.
  • uha_result_estrogen_dominant / uha_result_balanced — routes UHA-2 variant + LP link.
  • uha_backup_sent (UHA-1), uha_results_sent (UHA-2), uha_nudge1_sent / uha_nudge2_sent (UHA-3 / UHA-3b), uha_nowwhat_sent (UHA-4), uha_pitch1_sent (UHA-5), uha_pitch2_sent (UHA-6) — per-email send flags for idempotency.
  • challenge_pitch_clicked — set on Challenge-CTA click in any pitch email; may suppress further pitches.
  • challenge_purchased — if present, suppress UHA-5 and UHA-6 (they've already bought).
  • uha_quiz_abandoned — terminal; halts nudges after max retries.
3 Decision Gates
  • UHA-1 Backup (always fires) — parallel side path; generic "thanks for your purchase" confirmation regardless of quiz state. Idempotent via uha_backup_sent.
  • UHA-2 Results (Step 3, +few min) — fires on uha_quiz_done → 2 variants branched on result tag; links to matching LP.
  • UHA-3 Nudge #1 (+4h) — IF NOT uha_quiz_done → send catch-up + tag uha_nudge1_sent.
  • UHA-3b Nudge #2 (Day 2 · 9am) — IF NOT uha_quiz_done AND uha_nudge1_sent → send second catch-up + tag uha_nudge2_sent.
  • UHA-4 Now-What (Day 3 · 9am) — conditional content by result tag; soft Challenge reference.
  • UHA-5 Pitch #1 (Day 6 · 9am) — Challenge upsell. SUPPRESS if challenge_purchased.
  • UHA-6 Pitch #2 (Day 10 · 9am) — final Challenge upsell. SUPPRESS if challenge_purchased.
  • Nudge cap — max 2 quiz nudges; after Day 2 with no completion → apply uha_quiz_abandoned, but Now-What + Pitches still fire (generic content).
4 Exits & Errors
  • Refund — Stripe refund webhook → remove uha_purchased, add uha_refunded, halt workflow.
  • Challenge purchased mid-flow — on challenge_purchased, suppress UHA-5 and UHA-6 (don't pitch what they've bought).
  • Unsubscribe — GHL suppression handles delivery; workflow continues silently.
  • Quiz abandoned — terminal tag uha_quiz_abandoned after Day 2 + 2 nudges; Now-What/Pitches use generic (non-result) content.
  • Re-purchase — if contact re-enters with stale tags, clear uha_*_sent flags in the entry step before proceeding.
5 Open Questions

Confirm: if challenge_purchased fires mid-sequence, do we also suppress UHA-4 Now-What (Day 3) or only the two pitches? Also: should challenge_pitch_clicked (without purchase) dampen the second pitch's tone?
Refund / cancel exits — pattern not yet canonical.
Need a shared convention across UHA / Challenge / MEM / HWN. Current proposal: Stripe refund webhook → remove entry tag + add *_refunded + halt.
Quiz-abandoned fallback content.
For UHA-4/5/6, confirm the "generic" (no-result) content copy exists. Otherwise quiz abandoners see broken merge tags.

UHA Quiz Completion Sync — Interact Setup
This bridge turns a completed UHA quiz into usable automation state inside GHL. If this mapping breaks, UHA-2, UHA-4, and the no-result recovery logic all drift out of sync.
1
Create the GHL inbound webhook
In GHL, build the workflow that handles quiz completion and copy its inbound webhook URL. This is the endpoint Interact will POST to after a user submits the 48-question assessment.
2
Configure Interact result redirect + webhook
In Interact, wire the UHA quiz so submit sends contact data to GHL and then redirects to the correct results page. Keep the redirect branch aligned to the two live outcomes: estrogen_dominant and balanced.
3
Map payload fields cleanly
Required fields: email, first_name, quiz_name, result_type, and completion timestamp. Normalize result_type to the exact values GHL expects so downstream branches do not split incorrectly.
4
Write quiz-complete state back into GHL
On webhook receipt, add uha_quiz_done, set the result tag (uha_result_estrogen_dominant or uha_result_balanced), and store any custom value field used by the UHA-2/UHA-4 branch logic.
5
Guard the recovery path
Confirm the +4 hour decision step checks for completion before UHA-3 fires. The webhook should arrive fast enough that a real quiz finisher never falls into the nudge branch unless the sync failed.
6
Test each result end-to-end
Run two test submissions, one per outcome. Verify: correct result page loads, uha_quiz_done lands in GHL, the matching result tag is set, UHA-2 fires with the right variant, and UHA-3 stays suppressed.
