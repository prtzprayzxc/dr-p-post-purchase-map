HWN — Automation Notes

1 Workflow Context
Workflow
HWN | Onboarding (partial) → Monthly Cycle (not yet modeled)
Trigger
GHL checkout at /hormone-wellness-network-checkout (monthly $47) or /hormone-wellness-network-checkout-annual ($470/yr) → confirmation at /hormone-wellness-network-membership-thank-you-reg → tag hwn_active; Kajabi grants access; password branch on kajabi_account_created.
Entry
Contact has hwn_active AND NOT hwn_2_sent.
Delivery
Currently modeled: 2 onboarding emails (HWN-1 Welcome, HWN-2 Rhythm) + Kajabi system access. Not yet modeled: monthly masterclass reminders, replays, pre-renewal recaps, dunning.
Price
$47/mo OR $470/yr (annual saves ~17%).
2 Key GHL Tags
  • hwn_active (alias: purchased_hwn) — primary entry flag; removed on cancel. Also suppresses MEM-12/13/14 cross-sell.
  • hwn_term_monthly / hwn_term_annual — set by offer SKU; drives pre-renewal cadence (not yet modeled).
  • kajabi_account_created — persistent; drives password-setup vs direct-login language.
  • hwn_1_sent (Welcome), hwn_2_sent (Rhythm) — per-email send flags for the modeled onboarding.
  • Future (when monthly cycle is modeled): hwn_mc_reminder_sent_<yyyymm>, hwn_mc_replay_sent_<yyyymm>, hwn_prerenewal_sent_<yyyymm>, hwn_renewal_failed.
  • hwn_cancelled, hwn_refunded — terminal states.
3 Decision Gates
  • Steps 1-2 (Checkout + Confirmation) — GHL checkout at /hormone-wellness-network-checkout or /hormone-wellness-network-checkout-annual; on success → /hormone-wellness-network-membership-thank-you-reg; tag hwn_active applied; Kajabi access granted in parallel.
  • HWN-1 Welcome (Now) — primary onboarding email; password-setup branch on kajabi_account_created (new vs returning).
  • HWN-2 Rhythm (Day 2 · 9am) — how-to-get-the-most + community nudge.
  • Kajabi System (+ minutes) — side/system card: Kajabi creates user, fires setup-password email for new users (we don't author this send; Kajabi owns it).
  • Not yet modeled in visual: monthly masterclass reminder (T-3d), replay (T+1d), pre-renewal recap (monthly T-5d / annual T-14d), renewal-failure dunning.
4 Exits & Errors
  • Cancel — Kajabi subscription-cancel webhook → remove hwn_active, add hwn_cancelled, halt sends.
  • Refund (first month) — Stripe refund webhook → remove hwn_active, add hwn_refunded, halt.
  • Renewal failure (future) — on hwn_renewal_failed, trigger dunning sub-flow (to be modeled): recovery email → 3d wait → retry → 3 failures → hwn_cancelled.
  • Unsubscribe — suppression list; workflow continues silently.
